# Proposal: improve-ai-context

## Why

The course evaluates not only the app feature result, but also AI-assisted engineering evidence: prompt records, human decisions, build troubleshooting, IDE workflow, and reusable project context.

## What

- Maintain AI prompt records for requirement analysis, design, implementation, testing, and build troubleshooting.
- Keep AI-readable docs aligned with the evolving codebase.
- Add IDE + Bazel collaboration instructions for Trae or VS Code.
- Document build failures, fixes, query outputs, and stage retrospectives.
- Prepare a minimal build-helper or plugin plan if time allows.

## Impact

- Affects `docs/ai-context`, `openspec/prompt.md`, and per-change task evidence.
- Does not directly change app behavior.
- Supports final review and future AI collaboration.
