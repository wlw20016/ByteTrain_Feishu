# ByteTrain Feishu Landing Page Design

## Goal

Create a static landing page for the ByteTrain Feishu defense presentation. The page should introduce the product and engineering results from the complete defense script at `C:\Users\24155\Desktop\答辩\答辩文案(1).md`.

## Visual Direction

The approved direction is dark, minimal, and designed, with the visual language of a premium product showcase. It must avoid finance imagery. The hero uses floating engineering/product objects: a mobile app screen, command panel, module graph, and diagnostic context elements.

## Page Structure

1. Hero: project name, positioning, core value, and top metrics.
2. Product Flow: Messages and Mail mobile workflows, including list, detail, and pagination.
3. Two Phases: Android + Gradle quick prototype, then Bazel + VSCode engineering phase.
4. Highlights: OpenSpec, Bazel module management, VSCode plugin and AI diagnostic context.
5. Android Core: unified UI model mapping and dynamic page size strategy.
6. Engineering Track: Bazel, Rust SDK + Protobuf, VSCode helper, AI build diagnosis skill.
7. Demo Runbook: ordered defense demo flow and commands.
8. Footer: source file references and project identity.

## Implementation

Build a single static HTML file at `landing-page/index.html` with embedded CSS and small vanilla JavaScript for progress/navigation polish. No external dependencies are required.

## Verification

Open the page in the browser, check desktop and mobile layout, and verify key text from the defense script is represented clearly without overflow or finance-themed imagery.
